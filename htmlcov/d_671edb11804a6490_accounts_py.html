1""" 

2Account 

3 

4The Account represents the game "account" and each login has only one 

5Account object. An Account is what chats on default channels but has no 

6other in-game-world existence. Rather the Account puppets Objects (such 

7as Characters) in order to actually participate in the game world. 

8 

9 

10Guest 

11 

12Guest accounts are simple low-level accounts that are created/deleted 

13on the fly and allows users to test the game without the commitment 

14of a full registration. Guest accounts are deactivated by default; to 

15activate them, add the following line to your settings file: 

16 

17 GUEST_ENABLED = True 

18 

19You will also need to modify the connection screen to reflect the 

20possibility to connect with a guest account. The setting file accepts 

21several more options for customizing the Guest account system. 

22 

23"""

28class Account(DefaultAccount): 

29 """ 

30 This class describes the actual OOC account (i.e. the user connecting 

31 to the MUD). It does NOT have visual appearance in the game world (that 

32 is handled by the character which is connected to this). Comm channels 

33 are attended/joined using this object. 

34 

35 It can be useful e.g. for storing configuration options for your game, but 

36 should generally not hold any character-related info (that's best handled 

37 on the character level). 

38 

39 Can be set using BASE_ACCOUNT_TYPECLASS. 

40 

41 

42 * available properties 

43 

44 key (string) - name of account 

45 name (string)- wrapper for user.username 

46 aliases (list of strings) - aliases to the object. Will be saved to database as AliasDB entries but returned as strings. 

47 dbref (int, read-only) - unique #id-number. Also "id" can be used. 

48 date_created (string) - time stamp of object creation 

49 permissions (list of strings) - list of permission strings 

50 

51 user (User, read-only) - django User authorization object 

52 obj (Object) - game object controlled by account. 'character' can also be used. 

53 sessions (list of Sessions) - sessions connected to this account 

54 is_superuser (bool, read-only) - if the connected user is a superuser 

55 

56 * Handlers 

57 

58 locks - lock-handler: use locks.add() to add new lock strings 

59 db - attribute-handler: store/retrieve database attributes on this self.db.myattr=val, val=self.db.myattr 

60 ndb - non-persistent attribute handler: same as db but does not create a database entry when storing data 

61 scripts - script-handler. Add new scripts to object with scripts.add() 

62 cmdset - cmdset-handler. Use cmdset.add() to add new cmdsets to object 

63 nicks - nick-handler. New nicks with nicks.add(). 

64 

65 * Helper methods 

66 

67 msg(text=None, **kwargs) 

68 execute_cmd(raw_string, session=None) 

69 search(ostring, global_search=False, attribute_name=None, use_nicks=False, location=None, ignore_errors=False, account=False) 

70 is_typeclass(typeclass, exact=False) 

71 swap_typeclass(new_typeclass, clean_attributes=False, no_default=True) 

72 access(accessing_obj, access_type='read', default=False) 

73 check_permstring(permstring) 

74 

75 * Hook methods (when re-implementation, remember methods need to have self as first arg) 

76 

77 basetype_setup() 

78 at_account_creation() 

79 

80 - note that the following hooks are also found on Objects and are 

81 usually handled on the character level: 

82 

83 at_init() 

84 at_cmdset_get(**kwargs) 

85 at_first_login() 

86 at_post_login(session=None) 

87 at_disconnect() 

88 at_message_receive() 

89 at_message_send() 

90 at_server_reload() 

91 at_server_shutdown() 

92
