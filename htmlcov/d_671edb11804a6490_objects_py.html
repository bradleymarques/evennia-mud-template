4The Object is the "naked" base class for things in the game world.

13from evennia.objects.objects import DefaultObject

16class ObjectParent: 

17 """ 

18 This is a mixin that can be used to override *all* entities inheriting at 

19 some distance from DefaultObject (Objects, Exits, Characters and Rooms). 

20 

21 Just add any method that exists on `DefaultObject` to this class. If one 

22 of the derived classes has itself defined that same hook already, that will 

23 take precedence. 

24 

25 """

28class Object(ObjectParent, DefaultObject): 

29 """ 

30 This is the root typeclass object, implementing an in-game Evennia 

31 game object, such as having a location, being able to be 

32 manipulated or looked at, etc. If you create a new typeclass, it 

33 must always inherit from this object (or any of the other objects 

34 in this file, since they all actually inherit from BaseObject, as 

35 seen in src.object.objects). 

36 

37 The BaseObject class implements several hooks tying into the game 

38 engine. By re-implementing these hooks you can control the 

39 system. You should never need to re-implement special Python 

40 methods, such as __init__ and especially never __getattribute__ and 

41 __setattr__ since these are used heavily by the typeclass system 

42 of Evennia and messing with them might well break things for you. 

43 

44 

45 * Base properties defined/available on all Objects 

46 

47 key (string) - name of object 

48 name (string)- same as key 

49 dbref (int, read-only) - unique #id-number. Also "id" can be used. 

50 date_created (string) - time stamp of object creation 

51 

52 account (Account) - controlling account (if any, only set together with 

53 sessid below) 

54 sessid (int, read-only) - session id (if any, only set together with 

55 account above). Use `sessions` handler to get the 

56 Sessions directly. 

57 location (Object) - current location. Is None if this is a room 

58 home (Object) - safety start-location 

59 has_account (bool, read-only)- will only return *connected* accounts 

60 contents (list of Objects, read-only) - returns all objects inside this 

61 object (including exits) 

62 exits (list of Objects, read-only) - returns all exits from this 

63 object, if any 

64 destination (Object) - only set if this object is an exit. 

65 is_superuser (bool, read-only) - True/False if this user is a superuser 

66 

67 * Handlers available 

68 

69 aliases - alias-handler: use aliases.add/remove/get() to use. 

70 permissions - permission-handler: use permissions.add/remove() to 

71 add/remove new perms. 

72 locks - lock-handler: use locks.add() to add new lock strings 

73 scripts - script-handler. Add new scripts to object with scripts.add() 

74 cmdset - cmdset-handler. Use cmdset.add() to add new cmdsets to object 

75 nicks - nick-handler. New nicks with nicks.add(). 

76 sessions - sessions-handler. Get Sessions connected to this 

77 object with sessions.get() 

78 attributes - attribute-handler. Use attributes.add/remove/get. 

79 db - attribute-handler: Shortcut for attribute-handler. Store/retrieve 

80 database attributes using self.db.myattr=val, val=self.db.myattr 

81 ndb - non-persistent attribute handler: same as db but does not create 

82 a database entry when storing data 

83 

84 * Helper methods (see src.objects.objects.py for full headers) 

85 

86 search(ostring, global_search=False, attribute_name=None, 

87 use_nicks=False, location=None, ignore_errors=False, account=False) 

88 execute_cmd(raw_string) 

89 msg(text=None, **kwargs) 

90 msg_contents(message, exclude=None, from_obj=None, **kwargs) 

91 move_to(destination, quiet=False, emit_to_obj=None, use_destination=True) 

92 copy(new_key=None) 

93 delete() 

94 is_typeclass(typeclass, exact=False) 

95 swap_typeclass(new_typeclass, clean_attributes=False, no_default=True) 

96 access(accessing_obj, access_type='read', default=False) 

97 check_permstring(permstring) 

98 

99 * Hooks (these are class methods, so args should start with self): 

100 

101 basetype_setup() - only called once, used for behind-the-scenes 

102 setup. Normally not modified. 

103 basetype_posthook_setup() - customization in basetype, after the object 

104 has been created; Normally not modified. 

105 

106 at_object_creation() - only called once, when object is first created. 

107 Object customizations go here. 

108 at_object_delete() - called just before deleting an object. If returning 

109 False, deletion is aborted. Note that all objects 

110 inside a deleted object are automatically moved 

111 to their <home>, they don't need to be removed here. 

112 

113 at_init() - called whenever typeclass is cached from memory, 

114 at least once every server restart/reload 

115 at_cmdset_get(**kwargs) - this is called just before the command handler 

116 requests a cmdset from this object. The kwargs are 

117 not normally used unless the cmdset is created 

118 dynamically (see e.g. Exits). 

119 at_pre_puppet(account)- (account-controlled objects only) called just 

120 before puppeting 

121 at_post_puppet() - (account-controlled objects only) called just 

122 after completing connection account<->object 

123 at_pre_unpuppet() - (account-controlled objects only) called just 

124 before un-puppeting 

125 at_post_unpuppet(account) - (account-controlled objects only) called just 

126 after disconnecting account<->object link 

127 at_server_reload() - called before server is reloaded 

128 at_server_shutdown() - called just before server is fully shut down 

129 

130 at_access(result, accessing_obj, access_type) - called with the result 

131 of a lock access check on this object. Return value 

132 does not affect check result. 

133 

134 at_pre_move(destination) - called just before moving object 

135 to the destination. If returns False, move is cancelled. 

136 announce_move_from(destination) - called in old location, just 

137 before move, if obj.move_to() has quiet=False 

138 announce_move_to(source_location) - called in new location, just 

139 after move, if obj.move_to() has quiet=False 

140 at_post_move(source_location) - always called after a move has 

141 been successfully performed. 

142 at_object_leave(obj, target_location) - called when an object leaves 

143 this object in any fashion 

144 at_object_receive(obj, source_location) - called when this object receives 

145 another object 

146 

147 at_traverse(traversing_object, source_loc) - (exit-objects only) 

148 handles all moving across the exit, including 

149 calling the other exit hooks. Use super() to retain 

150 the default functionality. 

151 at_post_traverse(traversing_object, source_location) - (exit-objects only) 

152 called just after a traversal has happened. 

153 at_failed_traverse(traversing_object) - (exit-objects only) called if 

154 traversal fails and property err_traverse is not defined. 

155 

156 at_msg_receive(self, msg, from_obj=None, **kwargs) - called when a message 

157 (via self.msg()) is sent to this obj. 

158 If returns false, aborts send. 

159 at_msg_send(self, msg, to_obj=None, **kwargs) - called when this objects 

160 sends a message to someone via self.msg(). 

161 

162 return_appearance(looker) - describes this object. Used by "look" 

163 command by default 

164 at_desc(looker=None) - called by 'look' whenever the 

165 appearance is requested. 

166 at_get(getter) - called after object has been picked up. 

167 Does not stop pickup. 

168 at_drop(dropper) - called when this object has been dropped. 

169 at_say(speaker, message) - by default, called if an object inside this 

170 object speaks 

171 

172 """ 

173 

174 pass
